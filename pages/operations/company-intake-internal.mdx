# Company Intake: Internal Import Workflow

> **Internal use only.** This page maps answers from the customer-facing
> [Company Onboarding Questionnaire](/operations/company-intake) to the
> exact CMS tabs and database columns we paste them into. Do not share this
> page externally.

When a completed intake comes back from a customer, work through it in this
order. A clean fully-returned intake should take about 20 minutes to import.

---

## Field mapping

### Section 1: About your organisation

| Question | CMS field | DB column |
|---|---|---|
| 1.1 Registered company name | Settings → Company name | `companies.name` |
| 1.2 Display name | Settings → Short name | `companies.short_name` |
| 1.3 Website | Settings → Website | `companies.website` |
| 1.4 HQ location | Settings → Address | `companies.address`, `postcode` |
| 1.5 Countries covered | *(internal use, crisis-resource defaults)* | n/a |
| 1.6 Total headcount | Settings → Total employees | `companies.total_employees` |
| 1.7 Pilot cohort size | *(internal, used for uptake expectations)* | n/a |
| 1.8 Launch date | Set status `planning → launched` on this date | `companies.status` |
| 1.9 Trial duration | Renewal calendar reminder | n/a |
| 1.10 Success metrics | Pilot review template | n/a |

Generate the slug from 1.2 (e.g. "PwC Middle East" → `pwc-me`). Confirm
uniqueness before saving.

### Section 2: Contacts

| Question | CMS field | DB column |
|---|---|---|
| 2.1 HR / programme owner | Settings → Primary contact | `companies.contact_email`, `contact_phone` |
| 2.2 Welfare lead | Emergency Resources → Primary welfare contact | `emergency_contacts` JSONB |
| 2.3 IT contact | *(internal, Cognito, email deliverability)* | n/a |
| 2.4 MHFAs / mental health advocates | Organisational Structure → Offices → Mental Health Advocates (per office) | `companies.offices[].mentalHealthAdvocates` |

Advocates are now attached to the office they cover, mirroring the **Site Lead**
field, so Jodie can reassure an employee that there is a trained person at their
location. Each advocate takes a name, optional email/phone, and an optional
coverage note (e.g. "Tax team, floors 3-5"). For a company-wide first-aider with
no specific office, you can still add a card under Emergency Resources, but prefer
the per-office list where you know the location.

### Section 3: Org structure

| Question | CMS field | DB column |
|---|---|---|
| 3.1 Business units | Organisational Structure → Divisions | `companies.divisions` JSONB |
| 3.1 Departments under each | Organisational Structure → Departments | `companies.divisions[].departments` |
| 3.2 Offices | Organisational Structure → Offices | `companies.offices` JSONB |

If the customer gives you headcount per department, fill `employeeCount`
on each leaf. If they don't, leave it blank, the dashboard will use the
honest empty state (per the live-only fix).

### Section 4: Employee profile fields

This maps to the `companies.profile_fields` JSONB column (added in
migration 013). Edit via the **Employee fields** tab in the CMS (the editor
itself is titled "Employee questions").

| Question | Field key | Field type |
|---|---|---|
| 4.1 Business unit name | `division` | `select_from_company_divisions` |
| 4.2 Team / practice name | `department` | `select_from_company_departments` |
| 4.3 Seniority ladder | `seniority` | `select`, copy the customer's exact options |
| 4.4 Work mode options | `work_mode` | `select`, copy the customer's exact options |
| 4.5 Primary office label | `office_location` | `select_from_company_offices` |
| 4.6 Bespoke fields | new key per field | `select` or `text`, add to `fields[]` array |

**One decision per question.** The editor now exposes a single switch —
**Add to profile** — per question. Anything you add is asked of employees
*and* automatically available as a dashboard filter; there is no separate
"ask everyone" or "use as filter" toggle to set. Sensible defaults are
baked into the catalogue (`required: true` for division, department and
seniority; `showInCohortFilter: true` for everything except the free-text
`role` field), so you normally just choose which questions to include and
edit their wording/options.

**Location is one question.** Do not add separate Country or City questions.
`office_location` is the only location question — each office in the org
structure carries its own city and country, so the dashboard's cohort
builder derives Country and City filters from the chosen office. Country
and City are retired from the question catalogue; if an older config still
contains them the editor renders them with a "covered by Primary office,
safe to remove" note, but never offers them as new questions.

### Section 5: Sign-in

| Question | CMS field | DB column |
|---|---|---|
| 5.1 Primary domains | Settings → Allowed email domains | `companies.settings.allowed_email_domains[]` |
| 5.2 Contractor domains | append to the same list | same |
| 5.3 Block personal email? | *(Cognito rule)* | n/a |
| 5.4 SSO requirement | *(Cognito identity provider, separate ticket)* | n/a |

If you need an immediate non-DB override (testing, demos), also add to
`DOMAIN_OVERRIDES` in `mind-measure-pro-mobile/src/services/CompanyResolver.ts`.

### Section 6: Internal support services

| Question | CMS field | DB column |
|---|---|---|
| 6.1 EAP | Emergency Resources → **EAP** tab | `companies.settings.eap[]` JSONB |
| 6.2 HR support | Emergency Resources → HR | `emergency_contacts` JSONB |
| 6.3 Occupational health | Emergency Resources → Occupational Health | `mental_health_services` JSONB |
| 6.4 Manager escalation paragraph | Hierarchy of Help → step 1 description | `companies.hierarchy_of_help` JSONB |
| 6.5 Other wellbeing programmes | Emergency Resources → **EAP** tab (add a programme) | `companies.settings.eap[]` JSONB |

The dedicated **EAP** tab carries descriptive fields (tagline, description,
what they offer, how to access) alongside the contact details, so Jodie can
introduce a programme conversationally rather than just reciting a number, e.g.
*"did you know your company subscribes to OpenUp..."*. Use the **enabled** toggle
to control whether a programme is shared with Jodie. EAP entries are stored on
`companies.settings.eap` and merged on save, so they do not affect other settings
keys (allowed email domains, locale, welfare contact). This is a Pro-only area;
universities list campus-based services instead.

### Section 7: External crisis support

| Question | CMS field | DB column |
|---|---|---|
| External services list | Emergency Resources → External crisis services | `local_resources` JSONB (category: `crisis`) |

If the customer didn't list anything, leave the platform defaults in place
for their country (UK: Samaritans, Shout; US: 988; UAE: Estijaba 8001717,
etc., see `national_mental_health_resources` table).

### Section 8: Hierarchy of help

| Question | CMS field | DB column |
|---|---|---|
| Preferred escalation order | Hierarchy of Help tab | `companies.hierarchy_of_help` JSONB |

Default order is documented at [Hierarchy of Help](/operations/hierarchy-of-help).
Apply any customer overrides as steps in that JSONB structure.

### Section 9: Bespoke Jodie questions

| Question | CMS field | DB column |
|---|---|---|
| Each bespoke question | Institutional Agents → Custom questions | `institution_agents.custom_questions` JSONB |

Adding a custom question triggers Jodie regeneration via the
`/api/agents/create` endpoint. Verify the new prompt is live before
launch.

### Section 10: Privacy & reporting

| Question | Where it lives |
|---|---|
| 10.1 Who sees results | RBAC config, set role visibility in Settings |

| 10.3 Retention period | Settings → `companies.settings.retention_months` |
| 10.4 Reporting cadence | Wellbeing Reports → schedule |
| 10.5 Format | Wellbeing Reports → channels |
| 10.6 DPA | Legal, send standard DPA, file signed copy |
| 10.7 Data residency | *(internal, Aurora region routing)* |

### Section 11: Launch & training

| Question | Where it lives |
|---|---|
| 11.1 Distribution | iOS = TestFlight build; web = no action |
| 11.2 All-hands brief | Schedule with Comms 2 weeks before launch |
| 11.3 Admin training | Schedule with Customer Success 1 week before launch |
| 11.4 Comms owner | Add to onboarding tracker |
| 11.5 Dates to avoid | Block in launch calendar |

## Import order

1. **Create the company shell**, Superuser → Companies → New. Paste
   sections 1.1-1.6 and the slug.
2. **Org structure**, Organisational Structure tab. Paste section 3.1
   (divisions and their departments) and section 3.2 (offices). While you
   are here, add any per-office mental health advocates from section 2.4.
3. **Employee fields**, paste any customer overrides from
   section 4. If the customer didn't change anything, the corporate
   default from migration 013 is already in place. Use "Reset to
   recommended" in the editor to restore that default at any time.
4. **Allowed email domains**, Settings tab. Paste section 5.1 and 5.2 in.
5. **Emergency Resources**, paste sections 6.2, 6.3 and 7 into their cards,
   and sections 6.1 and 6.5 into the **EAP** tab (one entry per programme,
   with the descriptive fields completed so Jodie can offer them naturally).
6. **Hierarchy of Help**, paste section 8.
7. **Custom Jodie questions**, Institutional Agents tab. Paste section 9.
   Confirm Jodie regeneration succeeded.
8. **Privacy settings**, Settings tab. Apply section 10 answers.
9. **Smoke test**, sign in to the mobile app with a test email matching
   the allowed domain, complete the baseline, open the dashboard and
   confirm the cohort filters surface the right division/department lists.
10. **Status flip**, when the smoke test passes and the launch date is
    confirmed, move `status` from `planning` to `launched`.

## Brand assets (handled separately by Comms)

Brand assets are explicitly excluded from the customer questionnaire, the
customer's communications team handles logo files, brand colour and any
campus / hero imagery as a side workstream. When those arrive:

- Logo files → upload via Settings → Brand. Lands in S3
  (`mindmeasure-user-content-459338929203`).
- Primary / secondary colours → Settings → `companies.primary_color`,
  `secondary_color`.
- Hero image → optional, Settings → `companies.campus_image`.
