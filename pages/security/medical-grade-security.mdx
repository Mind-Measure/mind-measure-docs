# Medical-Grade Security Implementation

## Overview

Mind Measure has implemented comprehensive medical-grade security features to ensure HIPAA, GDPR, and SOC2 compliance. This implementation provides enterprise-level security for healthcare applications handling Protected Health Information (PHI).

> **Status note (June 2026).** This page describes the target security service layer. MFA (AWS Cognito), audit logging and RBAC are live. **Application-level field encryption is now partially in force**: the verbatim check-in transcript (`assessment_transcripts.transcript`) is encrypted at rest with KMS envelope encryption (per-record AES-256 data key, `enc:v1:` format) on the mobile write path and decrypted server-side only where the analysis pipeline needs it. Storage-layer at-rest protection (Aurora storage encryption and S3, AES-256 via AWS KMS) and TLS in transit remain in place underneath. The multimodal result blob (`fusion_outputs.analysis`) is **not** field-encrypted yet and sits in plaintext within the KMS-encrypted volume; that is a tracked defence-in-depth follow-up. See [Sub-processors & data flow](/security/sub-processors) and the [database reference](/architecture/database) for the authoritative current state.

## Security Architecture

### Multi-Factor Authentication (MFA)
- **AWS Cognito Integration**: Leverages AWS Cognito User Pools with MFA enabled
- **SMS-based MFA**: Primary authentication method using phone verification
- **TOTP Support**: Time-based One-Time Password using authenticator apps (Google Authenticator, Authy)
- **Strong Password Policies**: Minimum 12 characters with complexity requirements

### Comprehensive Audit Logging
- **Tamper-proof Logs**: All PHI access and modifications are logged immutably
- **Real-time Monitoring**: Critical security events trigger immediate alerts
- **Compliance Reporting**: Automated generation of HIPAA, GDPR, and SOC2 reports
- **Risk-based Classification**: Events categorized as LOW, MEDIUM, HIGH, or CRITICAL

### Field-Level Encryption (live for transcripts)

Application-level field encryption is enforced for the verbatim check-in transcript (`assessment_transcripts.transcript`). The mobile write path encrypts before INSERT and the core/pro-core read paths decrypt only where the analysis pipeline needs the text. Extending this to the `fusion_outputs.analysis` blob is a tracked follow-up.

- **KMS envelope encryption**: a per-record AES-256 data key from KMS `GenerateDataKey`, AES-256-GCM locally (authenticated encryption)
- **Self-describing format**: `enc:v1:<base64 JSON{k,iv,t,c}>`; values without the `enc:v1:` prefix are treated as legacy plaintext and pass through on read, so migration is safe
- **Key management**: a dedicated customer-managed KMS key (`alias/mindmeasure-phi`, eu-west-2) with annual rotation, usable only by the application service principal
- **PHI field scope**: the verbatim transcript today; the multimodal result blob is the next candidate

## Implementation Details

### MFA Service Configuration

```typescript
// Initialize MFA Service
const mfaService = createMFAService();

// Sign up with MFA requirement
const result = await mfaService.signUpWithMFA(
 email,
 password,
 phoneNumber
);

// Handle MFA challenge
const authResult = await mfaService.respondToMFAChallenge(
 challengeName,
 session,
 mfaCode,
 email
);
```

### Audit Logging Usage

```typescript
// Initialize Audit Logger
const auditLogger = createAuditLogger(databaseService);

// Log PHI access
await auditLogger.logPHIAccess(
 'PHI_VIEW',
 userId,
 userEmail,

);

// Log authentication events
await auditLogger.logAuth(
 'LOGIN_SUCCESS',

 userEmail,

);
```

## Database Schema

### Audit Logs Table

The audit logs table provides comprehensive tracking of all system activities:

```sql
CREATE TABLE audit_logs (
 id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
 timestamp TIMESTAMPTZ NOT NULL DEFAULT NOW(),
 user_id VARCHAR(255) NOT NULL,
 user_email VARCHAR(255),
 action VARCHAR(100) NOT NULL,
 resource VARCHAR(100) NOT NULL,
 resource_id VARCHAR(255),
 details JSONB,
 ip_address INET,
 user_agent TEXT,
 session_id VARCHAR(255),
 success BOOLEAN NOT NULL DEFAULT true,
 error_message TEXT,
 risk_level VARCHAR(20) NOT NULL,
 compliance_flags TEXT[]
);
```
