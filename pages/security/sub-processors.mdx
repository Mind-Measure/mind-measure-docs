## Roles

For institutional deployments the **institution is the data controller**, Mind
Measure Ltd is the **processor**, and the third parties below are
**sub-processors**. All personal data is stored and processed in UK/EU AWS
regions (`eu-west-2`); where a sub-processor operates outside the UK/EEA,
transfers rely on the safeguards described in the privacy policy (Standard
Contractual Clauses).

## What happens in one check-in

A check-in is **scored entirely on the user's device**; the fused score
(70% text, 15% audio, 15% visual) is computed client-side and only the derived
result is persisted. The raw modalities touch sub-processors as follows:

1. **Voice conversation -> ElevenLabs.** The app runs the conversation through
   the ElevenLabs React SDK (`@elevenlabs/react`). ElevenLabs streams the
   user's microphone audio, generates Jodie's spoken replies, and returns the
   running **transcript** (each turn's text) to the device. Speech-to-text is
   done inside ElevenLabs as part of the conversation; there is no separate
   transcription sub-processor.
2. **Transcript text -> AWS Bedrock.** The device sends the transcript to
   Bedrock (Claude 3.7 Sonnet) via `/api/bedrock/analyze-text` for the 70%
   text analysis. Bedrock returns linguistic features/scores; it does not
   retain inputs or train on them.
3. **Sampled video frames -> AWS Rekognition.** Frames captured during the
   check-in are sent to Rekognition for facial-landmark / expression features
   (the 15% visual signal). Frames are not persisted as raw media.
4. **Audio acoustic features -> on device.** The 15% audio signal (pitch,
   timing, energy) is extracted locally. The raw audio is not sent to a text
   model.
5. **Derived result -> AWS (Aurora + S3).** The fused score and features land
   in `fusion_outputs`; the transcript is stored best-effort in
   `assessment_transcripts`. Storage is encrypted Aurora Serverless v2; S3 is a
   temporary buffer for media per the retention policy.

## Live sub-processors

| Provider | Role | Personal data it receives | Region |
|---|---|---|---|
| **AWS** (Aurora, S3, Cognito, SES, CloudWatch) | Storage, auth, transactional email, logs | Account + profile data, derived scores/features, transcripts; auth tokens; operational logs | UK/EU (`eu-west-2`) |
| **AWS Bedrock** (Claude) | Check-in text analysis (70%) and back-office generation | Transcript text (check-in); aggregated/de-identified inputs (back-office) | EU |
| **AWS Rekognition** | Visual analysis (15%) | Sampled video frames during a check-in (not persisted as raw media) | EU |
| **ElevenLabs** | Conversational voice agent (Jodie) + in-conversation speech-to-text | Live microphone audio and the resulting conversation transcript | See ElevenLabs DPA |
| **Vercel** | Hosting + serverless API compute (front door) | Data in transit / ephemeral request handling; institutional dashboards serve aggregated, anonymised data | Global edge / configured region |
| **Sentry** | Error and performance monitoring | Operational telemetry; may incidentally capture limited context in error payloads | Per Sentry DPA |

Back-office generation (AI Insights, weekly reflections, cohort insight, report
narratives) uses **AWS Bedrock (Claude Haiku 4.5)** and operates on
aggregated, k-anonymised data (cohort floor of five), not raw individual PHI.
See [Stack facts](/architecture/stack-facts) for the exact model identifiers.

## Explicitly NOT sub-processors

<Callout type="warning">
  These appear in older documentation but are **not in the live data path.**
  Do not list them as sub-processors.
</Callout>

- **OpenAI.** No live data flow, and now **fully decommissioned** (June 2026).
  Speech-to-text is done by ElevenLabs, not OpenAI Whisper. The legacy
  `mind-measure-scoring` Serverless service held two OpenAI GPT-4o-mini callers
  (`analyze-text` baseline extraction and a never-deployed `finalize-session-worker`),
  but it was dead (zero invocations over 90 days, no live caller in the app, which
  scores entirely client-side via Bedrock and Rekognition). Verified, then removed:
  the dead Lambdas, the SQS queue, the `/mindmeasure/openai/api_key` SSM parameter
  and the in-repo source were all deleted (the API key itself should be revoked in
  the OpenAI dashboard to finish). There is no remaining OpenAI code path. (The
  stale "Whisper / transcription" sub-processor row in the legal privacy policies
  was reconciled to ElevenLabs on 13/06/2026 to match this page.)
- **Supabase.** Fully migrated to AWS Aurora Serverless v2. No Supabase service
  processes Mind Measure data.
